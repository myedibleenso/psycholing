Rule transfer in biomedical event extraction¶

Event Extraction is concerned with identifying the relationship between entities.

The majority of events in the biomedical domain are biochemical reactions involving one or more participants, typically in an agent-patient construction.

Pattern-based rules for identifying textual mentions of such events can be generated automatically by extracting the shortest syntactic dependency path between known predicate-argument pairs.

This work examines the effects of applying rules generated for one event to a set of related events for which annotations are unavailable or incomplete.

Questions to be answered¶

  • Are a class of semantically related events necessarily syntactically similar?

    • Can we use patterns describing variations of the syntactic structure of one event to generalize rules for a class of related events?
    • How might one use knowledge of one event (i.e. the information encoded in a rule, including # of args, their semantic types, etc) to bootstrap a grammar for another (sparsely represented) event?

  • Is there a simple predictive model to estimate performance on transfer (ex. is it a function of path length, the final hop in the path, etc)?

  • How is path transfer related to trigger (predicate) diversity?

  • How should rules be combined for $n$-ary events?

    • This might be out of the scope of this work...

Background¶

Rules¶

The rules used in this work follow Odin-style rules (http://www.anthology.aclweb.org/P/P15/P15-4.pdf#page=139). Event rules are composed of some predicate (i.e. trigger) consisting of one or more tokens. Rules can be specified as a pattern over tokens or a pattern of syntactic dependencies. The output of a rule is a textual mention, a labeled collection of text spans.

Event rules¶

Rules for events specify predicates (i.e. triggers) using a token pattern. These tokens may be specified as a regular expression over any combination of the raw text, lemma, or PoS tag attibutes of the token.

An event rule may describe each of its arguments as a syntactic dependency path originating from the predicate.
Syntactic dependencies describe syntactic head $\leftrightarrow$ dependent relationships between words. Hops are directed, pointing from head to dependent. Patterns may follow hops in whichever direction is desired (head $\rightarrow$ dependent vs dependent $\rightarrow$ head). Each "hop" in the path leads to a word which may be constrained to varying degrees of lexicalization using a token pattern (described previously).

Each argument has a semantic type (ex. Protein vs Gene) and can consist of multiple tokens.

A note about event extraction¶

For this work, the entities (i.e. mentions that can serve as event arguments) are provided to us. In event extraction, our focus is only the relationship/interaction between entities (or other events).

Formulations¶

Trigger relevance to an event¶

I will define the relevance of a trigger $t_i$ to an event $e$ as its normalized frequency within that event category:

$Rel(t_i,e) = \frac{f(t_i,e)}{\sum_{j}f(t_j,e)}$

where $f(t_i,e)$ is the frequency of trigger $t_i$ for the event $e$

Relevance of a rule to an event¶

The relevance of a rule ($r$) for an event ($e$) should be a function of the relevance of a trigger ($t$) to $e$, the valency of the event ($v_e$), and the frequency of a rule ($r_f$):

$Rel(r, e) \approx f(Rel(t,e), v_e, r_f)$

Alternative formulation¶

Of the occurrences of argument for event ($a_{e}$), what percentage is covered by rule $r$?

$Rel(r, e) \approx \frac{r_{f}}{a_{e}}$

Experiment¶

Main idea¶

Take the most frequent event in training, extract paths (rules), and apply those rules to other events.

Details¶

Annotated corpora exist for identifying an events trigger (i.e. predicate) and its arguments.
In this work, Im going to focus on a class of events defined in the BioNLP 2009 GE task (i.e. Simple Events). This data is divided into three partitions: training, development, and testing.

From the training data, I will generate rules for the most commmon event and apapt those rules to a set of related events by substituting an appropriate predicate for each event while leaving the syntactic path unmodified:

Extracted Rule¶

pattern: |
  trigger=[lemma='phosphorylation']       
  Theme:Protein >prep_of

Adapted Rule¶

label: Ubiquitination 
pattern: |
  # the trigger has been changed
  trigger=[lemma='ubiquitination']       
  Theme:Protein >prep_of

In this work, I will attempt to use wholly unlexicalized paths for event arguments meaning that each hop in the path is unconstrained in terms of the identity of the word:

Lexicalized path¶

This lexicalized path originating from the predicate ate points to "Ayu" in the sentence: "Ayu's cats ate the lizard."

>nsubj [lemma=cat] >poss [tag=NNP]

Unlexicalized path¶

This unlexicalized path originating from the predicate ate points to "Ayu" in the sentence: "Ayu's cats ate the lizard."

>nsubj >poss

Evaluation¶

These rules will be evaluated against the development data. I will compare performance of the adapted rules on the development partition with a baseline measure where for each event, rules are extracted in the training data and then applied to all instances of that corresponding event in the development data.

from collections import defaultdict, Counter
import re
import numpy as np
import pandas as pd
pd.set_option('display.mpl_style', 'default') # Make the graphs a bit prettier


class EventRule(object):
    def __init__(self, rule_string):
        assert("trigger = " in rule_string)
        # find trigger, args, and hops
        self._process_rule(rule_string)
        # TODO: estimate rule contribution to Class c
        self.score = None

    def remove_whitespace(self, text):
        return re.sub(r"\s+", "", text)
        
    def _process_rule(self, rule_string):
        # find label
        label = re.search("label:\s*(?P<label>.*)", rule_string)
        # find count
        cnt = re.search("count:\s*(?P<count>.*)", rule_string)
        # find pattern
        pattern = re.findall("pattern: [|]\n\s*(.*)", rule_string, re.DOTALL | re.MULTILINE)[0]
        pattern = re.sub("\s+", " ", pattern).strip()
        m = re.search("(?P<trigger>^.*?(\s*\[.*?\])+)\s+(?P<argname>.*?(?=:)):(?P<argtype>.*?\s)\s*=\s(?P<path>.*)", pattern)
        self.trigger = m.group("trigger").strip()
        # attempt to extract trigger tags
        tt = re.findall("tag=(?P<tag>[^\s\]]+)", self.trigger)
        self.trigger_tags = [t.strip() for t in tt]
        # remove whitespace in label
        self.label = self.remove_whitespace(label.group("label"))
        # store count as int
        self.count = int(cnt.group("count"))
        # split on the whitespace preceding the edge direction specifier
        self.path = re.split(" (?=[<>])", m.group("path"))
        self.path_length = len(self.path)
        # store arg and semantic type of arg separately 
        self.argname = self.remove_whitespace(m.group("argname"))
        self.argtype = self.remove_whitespace(m.group("argtype"))
        self.arg = "{0}:{1}".format(self.argname, self.argtype)
        shortest_path = [self.trigger] + self.path + [self.arg]
        self.hops = zip(shortest_path, shortest_path[1:])

rf = open("generated_rules_with_trigger_tags.yml", "r").read()
rules = re.findall('- name:.*?(?=^$)', rf, re.DOTALL | re.MULTILINE)
# create event rule representations for any rule with a trigger
event_rules = [EventRule(rs) for rs in rules if "trigger = " in rs]
er = defaultdict(list)
# examine rules per (arg, path) pair
for e in event_rules:
    er[(e.arg, tuple(e.path))].append(e)
